# Client Onboarding Module

## Overview

The Client Onboarding module uses a Kanban-style board to track prospective clients through a 10-stage pipeline from initial contact to service activation.

---

## Pipeline Stages

| Stage | Name                   | Owner              | Description                             |
| ----- | ---------------------- | ------------------ | --------------------------------------- |
| 1     | Reach Out              | Operations Manager | Initial contact with prospective client |
| 2     | Assessment             | Supervisor         | Initial needs assessment                |
| 3     | Schedule Appointment   | Staff/Supervisor   | Book formal assessment meeting          |
| 4     | Assessment Report      | Supervisor         | Document findings and recommendations   |
| 5     | Clinical Authorization | Clinical Director  | Clinical approval (required gate)       |
| 6     | Assign Caregiver       | Supervisor         | Match appropriate caregiver             |
| 7     | Health Check           | Staff              | Caregiver health verification           |
| 8     | Health Assessment      | Clinical Director  | Formal caregiver health assessment      |
| 9     | 1-on-1 Assessment      | Supervisor         | Caregiver-client compatibility check    |
| 10    | Contract Start         | Operations Manager | Activate service agreement              |

---

## Using the Kanban Board

### Viewing the Board

- Access via **Onboarding** in main navigation
- Cards are displayed in columns by stage
- Drag cards between stages (permission-dependent)

### Card Information

Each card shows:

- Client name
- Days in current stage
- Assigned owner
- Quick actions

### Moving Cards

1. Drag card to new stage column
2. Add transition notes if required
3. Confirm the move
4. Relevant stakeholders are notified

### Stage 5 Gate

Clinical Director authorization is **required** before moving past stage 5. Cards cannot be dragged to stage 6 without approval.

---

## Card Details

Click a card to view/edit:

- Client information
- Contact details
- Stage history
- Attached documents
- Notes

### Adding Documents

1. Open card details
2. Click **Add Document**
3. Upload file
4. Document is attached to current stage

---

## Notifications

Stage transitions trigger notifications:

- New stage owner is notified
- Previous owner receives confirmation
- Clinical Director notified when cards reach stage 5

---

## Audit Trail

All stage transitions are logged with:

- Timestamp
- User who made the change
- From/to stages
- Notes added

View history in card details under **Activity**.
